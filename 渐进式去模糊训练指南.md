## 核心思想

### 问题场景

在实际应用中（如煤矿监控、监控摄像头等），输入图像往往存在模糊问题：
- **运动模糊**：物体或相机运动导致
- **散焦模糊**：对焦不准导致
- **环境因素**：灰尘、烟雾等导致

传统方法的问题：
1. **单帧去模糊**：信息不足，容易产生伪影
2. **RF（辐射场）去模糊**：依赖多视角互补模糊，遇到"所有图模糊方向相似"就失效

### 渐进式去模糊解决方案

我们提出的解决方案是**让去模糊网络和检测器互相当老师**，循环迭代：

```
初始模糊图像
    ↓
[Stage 1: 粗去模糊]  ← 去模糊网络进行初步去模糊
    ↓
RT-DETR特征提取
    ↓
[Stage 2: RF引导]  ← 使用检测器特征作为"RF"引导，优化去模糊
    ↓
更清晰的图像  ←→ 更准确的检测
    ↓
重复迭代，两边一起变强
```

### 核心优势

1. **渐进式训练**：先简单去模糊，再逐步优化
2. **协同优化**：去模糊网络和检测器互相促进
3. **RF引导**：利用检测器的多尺度特征引导去模糊（类似NeRF的多视角信息）
4. **无需GT清晰图像**：在推理时可以直接使用

## 训练流程

### 训练阶段

我们的训练分为三个阶段：

#### Stage 1: 预热阶段 (0 - warmup_iters)
- **去模糊网络**：冻结，使用简单去模糊预处理
- **RT-DETR**：训练，使用去模糊后的图像
- **目的**：让检测器先学会基本的检测能力

#### Stage 2: 联合训练阶段 (joint_train_start - rf_guidance_start)
- **去模糊网络**：解冻，开始训练
- **RT-DETR**：继续训练
- **交互**：去模糊网络的输出 → RT-DETR的输入
- **目的**：两个网络协同优化

#### Stage 3: RF引导阶段 (rf_guidance_start - 结束)
- **去模糊网络**：继续训练，接收RF引导
- **RT-DETR**：继续训练，提供特征作为引导
- **RF引导**：RT-DETR的编码器特征 → 投影后作为去模糊网络的引导
- **目的**：利用检测器的特征能力进一步提升去模糊质量

### 训练循环

每个训练步骤：

```python
# 1. 获取模糊图像
blur_img = batch['blur_img']

# 2. 根据训练阶段处理
if stage == 'warmup':
    # 预热：仅使用去模糊网络（冻结）
    with torch.no_grad():
        deblurred = deblur_net(blur_img)

elif stage == 'joint':
    # 联合：训练去模糊网络
    deblurred = deblur_net(blur_img)
    deblur_loss = compute_deblur_loss(deblurred, sharp_img)
    deblur_loss.backward()

elif stage == 'rf_guidance':
    # RF引导：使用检测器特征
    deblurred = deblur_net(blur_img)
    features = rtdetr.encoder(deblurred)
    guidance = project_features_to_guidance(features)
    deblurred_refined = deblur_net(blur_img, guidance=guidance)

# 3. 训练检测器
outputs = rtdetr(deblurred)
det_loss = criterion(outputs, targets)
det_loss.backward()
```

## 配置说明

### 去模糊网络配置

```yaml
deblur_net:
  type: DeblurWrapper
  deblur_net_type: simple      # 'simple' (轻量) or 'nafnet' (高质量)
  in_channels: 3
  out_channels: 3
  num_blocks: [1, 1, 1, 1]     # 每层的块数（影响参数量）
  channels: [32, 64, 128, 256] # 特征通道数
  drop_out_rate: 0.
  use_guidance: False          # 是否使用引导图像
  warmup_iters: 1000           # 预热迭代次数
  enable_gradual: True         # 渐进式训练
```

**参数建议**：
- **轻量级**（快速训练）：
  ```yaml
  num_blocks: [1, 1, 1, 1]
  channels: [32, 64, 128, 256]
  ```
- **高质量**（更慢但更好）：
  ```yaml
  num_blocks: [2, 2, 2, 2]
  channels: [64, 128, 256, 512]
  ```

### 训练阶段配置

```yaml
warmup_iters: 1000           # Stage 1结束的迭代数
joint_train_start: 2000      # Stage 2开始的迭代数
rf_guidance_start: 5000      # Stage 3开始的迭代数
use_rf_guidance: True         # 是否启用RF引导
deblur_loss_weight: 0.5      # 去模糊损失权重（相对于检测损失）
```

**调整建议**：

| 场景 | warmup_iters | joint_train_start | rf_guidance_start | deblur_loss_weight |
|------|--------------|-------------------|-------------------|-------------------|
| 小数据集（<1K图） | 500 | 1000 | 2000 | 0.3 |
| 中等数据集（1K-10K） | 1000 | 2000 | 5000 | 0.5 |
| 大数据集（>10K） | 2000 | 5000 | 10000 | 0.7 |

### 去模糊优化器配置

```yaml
deblur_optimizer:
  type: AdamW
  lr: 1e-4                   # 学习率（建议比检测器小）
  weight_decay: 1e-4
  betas: [0.9, 0.999]
```

### 数据增强配置

```yaml
train_dataloader:
  dataset:
    transforms:
      ops:
        # 添加合成模糊
        - {type: SyntheticBlur,
           blur_prob: 0.5,           # 50%的图像添加模糊
           blur_types: ['gaussian', 'motion', 'defocus']}
        # ... 其他增强
  collate_fn: deblur_collate_fn  # 重要！使用去模糊专用的collate
```

**模糊类型说明**：
- `gaussian`：高斯模糊（模拟散焦）
- `motion`：运动模糊（模拟运动）
- `defocus`：散焦模糊（模拟对焦不准）

## 高级用法

### 1. 自定义模糊核

如果需要模拟特定类型的模糊（如煤矿中的运动模糊）：

创建自定义模糊变换：

```python
# src/data/custom_blur.py
class CustomMotionBlur:
    def __init__(self, blur_prob=0.5, direction='horizontal'):
        self.blur_prob = blur_prob
        self.direction = direction

    def __call__(self, img, target=None):
        if random.random() < self.blur_prob:
            # 应用自定义运动模糊
            img = self.apply_motion_blur(img, self.direction)
        return img, target
```

在配置中使用：

```yaml
transforms:
  ops:
    - {type: CustomMotionBlur, blur_prob: 0.7, direction: 'horizontal'}
```

### 2. 使用真实模糊-清晰对

如果有标注的模糊图像：

```python
# src/data/blur_dataset.py
class PairedBlurDataset(torch.utils.data.Dataset):
    def __init__(self, blur_root, sharp_root, ann_file):
        self.blur_root = blur_root
        self.sharp_root = sharp_root
        # ...

    def __getitem__(self, idx):
        blur_img = Image.open(os.path.join(self.blur_root, self.blur_list[idx]))
        sharp_img = Image.open(os.path.join(self.sharp_root, self.sharp_list[idx]))
        return (blur_img, sharp_img), target
```

### 3. 调整RF引导策略

默认的RF引导使用简单的特征投影。可以自定义更复杂的引导策略：

```python
# src/solver/deblur_solver.py

class DeblurDetSolver(BaseSolver):
    def _get_rf_guidance(self, features):
        """自定义RF引导策略"""
        # 多尺度特征融合
        feat = torch.cat([
            F.interpolate(f, scale_factor=2**(i), mode='bilinear')
            for i, f in enumerate(features)
        ], dim=1)

        # 使用注意力机制
        guidance = self.attention_proj(feat)

        return guidance
```

### 4. 课程学习

逐步增加模糊程度：

```python
# src/data/curriculum_blur.py
class CurriculumBlur:
    def __init__(self, max_epochs=100):
        self.max_epochs = max_epochs

    def get_blur_intensity(self, epoch):
        # 从0.5逐渐增加到2.0
        return 0.5 + 1.5 * (epoch / self.max_epochs)

    def __call__(self, img, epoch):
        intensity = self.get_blur_intensity(epoch)
        return apply_blur(img, intensity)
```

## 常见问题

### Q1: 训练很慢怎么办？

**原因**：去模糊网络增加了计算量。

**解决方案**：
1. 使用更轻量的去模糊网络：
   ```yaml
   num_blocks: [1, 1, 1, 1]
   channels: [32, 64, 128, 256]
   ```
2. 减小batch size，但累积更多梯度：
   ```yaml
   batch_size: 2
   optimizer: {accum_iter: 4}  # 等效batch_size=8
   ```
3. 使用混合精度训练：
   ```bash
   python tools/train.py -c config.yml --amp
   ```

### Q2: 去模糊效果不好？

**原因**：可能是训练阶段设置不合理。

**检查清单**：
- [ ] 预热阶段是否太长？（尝试减少warmup_iters）
- [ ] 去模糊损失权重是否合适？（尝试调整deblur_loss_weight）
- [ ] 是否启用了RF引导？（use_rf_guidance: True）
- [ ] 数据增强的模糊类型是否合理？

### Q3: 检测精度下降了？

**原因**：去模糊网络可能过度平滑图像，损失细节。

**解决方案**：
1. 降低去模糊损失权重：
   ```yaml
   deblur_loss_weight: 0.2  # 从0.5降低到0.2
   ```
2. 使用更温和的模糊增强：
   ```yaml
   - {type: SyntheticBlur, blur_prob: 0.3, blur_types: ['gaussian']}
   ```
3. 在RF引导阶段增加检测器损失的权重。

### Q4: 内存不足（OOM）？

**原因**：去模糊网络和检测器同时训练消耗大量内存。

**解决方案**：
1. 使用梯度检查点：
   ```python
   # 在模型定义中
   from torch.utils.checkpoint import checkpoint
   def forward(self, x):
       return checkpoint(self._forward, x)
   ```
2. 减小输入分辨率：
   ```yaml
   train_dataloader:
     dataset:
       transforms:
         - {type: Resize, size: [512, 512]}  # 从640降到512
   ```
3. 使用梯度累积：
   ```yaml
   optimizer: {accum_iter: 2}
   ```

### Q5: 如何评估去模糊效果？

创建评估脚本：

```python
# tools/eval_deblur.py
import torch
from src.zoo.rtdetr.deblur_module import DeblurWrapper
from PIL import Image

# 加载模型
deblur_net = torch.load('checkpoint.pth')['deblur_net']
deblur_net.eval()

# 加载图像
blur_img = Image.open('blur.jpg')
blur_tensor = to_tensor(blur_img).unsqueeze(0)

# 去模糊
with torch.no_grad():
    sharp_tensor = deblur_net(blur_tensor)
    sharp_img = to_pil(sharp_tensor.squeeze(0))

# 保存
sharp_img.save('deblurred.jpg')
```

## 总结

渐进式去模糊训练通过让去模糊网络和检测器协同优化，实现了：

1. **更好的鲁棒性**：处理真实场景中的模糊图像
2. **端到端训练**：无需单独预训练去模糊网络
3. **RF引导**：利用检测器的多尺度特征提升去模糊质量
